# 🚀 คู่มือการสร้างและตั้งค่าระบบฐานข้อมูล LMDS ขั้นสมบูรณ์ (Complete Setup Guide)

เอกสารนี้รวบรวม **"ทุกข้อสงสัย ทุกขั้นตอน และโครงสร้างคอลัมน์"** ของฐานข้อมูล Logistics Master Data System (LMDS) ในรูปแบบที่รัดกุมและใช้งานตามแนวทางแบบ Enterprise 

## 🏗️ 1. โครงสร้าง Google Sheets (Database Schema)

เพื่อรองรับฟีเจอร์ทั้งหมด โปรเจคนี้จำเป็นต้องมี Sheet ทั้งหมด **6 หน้า** (ตั้งชื่อให้ตรงเป๊ะทุกตัวอักษร)

### 1️⃣ Sheet: `Database` (ตารางหลักสำหรับเก็บข้อมูลลูกค้า/สถานที่)
นี่คือหัวใจของระบบ มีไว้เพื่อเป็น Single Source of Truth ห้ามให้ใครมาลบข้อมูลเล่นเด็ดขาด ต้องซ่อนหรือจัดการสิทธิ์ให้ดี
ข้อมูลจะมีทั้งหมด 29 คอลัมน์ (ตาม `Config.gs`)

| คอลัมน์ | ชื่อหัวตาราง (Header) | คำอธิบาย (Description) |
|:---|:---|:---|
| A | **NAME** | ชื่อลูกค้าหรือร้านค้าต้นทาง (จาก SCG) |
| B | **LAT** | ละติจูด (หากยังไม่ยืนยัน ระบบจะหามาให้) |
| C | **LNG** | ลองจิจูด |
| D | SUGGESTED | ชื่อแนะนำ (กรณีจะเปลี่ยนชื่อ) |
| E | CONFIDENCE | ความมั่นใจของพิกัด/ข้อมูล |
| F | NORMALIZED | ชื่อที่ระบบตัดคำขยะออกแล้ว (สำหรับค้นหา) |
| G | VERIFIED | ติ๊กถูก `TRUE/FALSE` (ถ้าเป็น TRUE แปลว่าข้อมูลนี้สะอาดและยืนยันแล้ว) |
| H | SYS_ADDR | ที่อยู่เดิมที่ติดมาจากระบบบริษัท |
| I | GOOGLE_ADDR | ที่อยู่ที่ได้จาก Google Maps API |
| J | DIST_KM | ระยะห่างจากศูนย์ (กิโลเมตร) |
| K | **UUID** | [สำคัญมาก] รหัสระบุตัวตน 36 หลัก (ห้ามซ้ำห้ามสลับ) |
| L | PROVINCE | จังหวัด |
| M | DISTRICT | อำเภอ |
| N | POSTCODE | รหัสไปรษณีย์ |
| O | **QUALITY** | คะแนนคุณภาพข้อมูล (0-100) วิ่งออโต้ |
| P | **CREATED** | วันที่สร้าง (Timestamp) |
| Q | **UPDATED** | วันที่แก้ไขล่าสุด (Timestamp) |
| R | Coord_Source | แหล่งที่มาของพิกัด (เช่น Driver, SCG, Google) |
| S | Coord_Confidence | ความแม่นยำพิกัด (0-100) |
| T | Coord_Last_Updated| วันที่พิกัดเปลี่ยนล่าสุด |
| U | **Record_Status** | สถานะของข้อมูล (`Active`, `Inactive`, `Merged`) |
| V | Merged_To_UUID | ถ้ารวมชื่อซ้ำ ให้เอา UUID ตัวหลักมาใส่ช่องนี้ |
| W | Canonical_Name | ชื่อมาตรฐานที่ผ่านการทำปัดคำ (ลบบจก,วงเล็บ) |
| X | Canonical_Address| ที่อยู่มาตรฐานที่ปรับคำแล้ว (ลบ ซอย, หมู่) |
| Y | Name_FP | ลายนิ้วมือของชื่อ (Hash) |
| Z | Addr_FP | ลายนิ้วมือของที่อยู่ (Hash) |
| AA| Lat_Norm | พิกัดดัดแปลงให้เหลือ 6 ทศนิยม |
| AB| Lng_Norm | พิกัดดัดแปลงให้เหลือ 6 ทศนิยม |
| AC| Geo_Hash | ก้อนพิกัด 1 ตารางกริด (เอาไว้หาของซ้ำแบบเร็ว) |

### 2️⃣ Sheet: `NameMapping` (ตารางพจนานุกรมชื่อบริษัท)
ใช้บอกระบบและ AI ว่า "ชื่อแปลกๆ" นี้ ให้ชี้ไปที่ UUID ของบริษัทแม่ตัวไหน
*   A: `Variant_Name` (ชื่อที่เขียนผิด หรือเขียนย่อ)
*   B: `Master_UID` (UUID ของตัวจริงใน Database)
*   C: `Confidence_Score` (100 = คนรัยรอง, <100 = AI ทำ)
*   D: `Mapped_By` (ชื่อคน หรือ `AI_Agent`)
*   E: `Timestamp`

### 3️⃣ Sheet: `DQ_Review_Queue` (คิวตรวจสอบคุณภาพข้อมูลซ้ำซ้อน)
ใช้สำหรับให้คนมานั่งพิจารณาว่า สิ่งที่ระบบมองว่า "ซ้ำ" หรือ "Conflict" ทั้ง 8 ข้อ จะให้ทำยังไง
*   A: `Timestamp`, B: `Case_ID`, C: `Candidate_UUID_1`, D: `Candidate_UUID_2` 
*   E: `Name_1`, F: `Name_2`, G: `Canonical_Name_1`, H: `Canonical_Name_2`
*   ... ไปจนถึง P: `Review_Note` (ตาม `Config.gs`)

### 4️⃣ Sheet: `GPS_Queue` (คิวพิจารณาพิกัดคนขับที่ไม่ตรง)
เมื่อคนขับปักพิกัดห่างจาก Master แวะมาตรงนี้
*   A: `Timestamp`, B: `ShipToName`, C: `UUID_DB`, D: `LatLng_Driver`, E: `LatLng_DB`, F: `Diff_Meters`, G: `Reason`, H: `Approve` (ติ๊กกล่อง), I: `Reject` (ติ๊กกล่อง)

### 5️⃣ Sheet: `SCGนครหลวงJWDภูมิภาค` หรือชีตนำเข้าใดๆ
ชีตรับข้อมูลขาเข้าจาก External API

### 6️⃣ Sheet: `Archive_DB` (ใหม่)
สร้างไว้สำหรับเก็บข้อมูล `Inactive` (ที่ทำใหม่ในแผนนี้) หัวตารางให้ Copy จากชี้ต `Database` มาวางได้เลยทั้งหมด 29 คอลัมน์

## 💻 2. การทำงานของปุ่มคำสั่ง (Menu & UI Actions)

ระบบจะมีเมนูให้คุณกดด้านบนของ Google Sheets ชื่อว่า **"🚛 LMDS Menu"**
(บางเมนูจะถูกซ่อนไว้สำหรับแอดมินเท่านั้น โดยการเช็ค Email ผู้คลิก)

**วิธีกดใช้งานและการแก้ไขปัญหา (Troubleshooting):**

1.  **🚀 ตรวจสอบสุขภาพระบบ (System Health Check)**
    *   **ใช้งานเมื่อ:** เพิ่งลงโค้ดเสร็จใหม่ๆ หรือสร้างชีตใหม่
    *   **แก้ปัญหา:** เช็คว่าตั้งชื่อ Sheet ผิดไหม ขาดคอลัมน์สำคัญหรือเปล่า และเช็ค API Key พร้อมใช้ไหม ถ้าผ่านจะมีป๊อปอัปบอกว่า "System Integrity: OK"

2.  **📥 ซิงค์ข้อมูลจาก SCG (Sync New Data)**
    *   **ใช้งานเมื่อ:** ข้อมูลจากฝั่ง API ไหลเข้ามาที่ชีต `SCGนครหลวงJWDภูมิภาค` ต้องการอัปเดตเอาเข้า Master Database
    *   **สิ่งที่ระบบทำ:** มันจะหา "ข้อ 1.ชื่อเป๊ะ", "ข้อ 4.ชื่อสะกดไม่เหมือนแต่อนุโลม (AI mapping)" ถ้าใหม่กิ๊กจะสร้างแถวให้ ถ้าชื่อซ้ำแต่พิกัดไม่เหมือน (ข้อ 7) จะโดนโยนเข้า `GPS_Queue`

3.  **🧹 วิเคราะห์หาข้อมูลซ้ำซ้อน 8 ข้อยาก (Data Quality Full Check)**
    *   **ใช้งานเมื่อ:** อยากทำความสะอาดบ้านครั้งใหญ่ 
    *   **สิ่งที่ระบบทำ:** ระบบจะประมวลผล GeoHash และ Matching อัลกอริทึม หากมันเจอข้อ 6 (สาขาเดียวกันแต่คนละพิกัด) ข้อ 8 (คนละชื่อพิกัดเดียวกันเป๊ะเหมือนสแปม) ระบบจะไม่รวมมันไว้ด้วยกัน (เพราะมันอันตรายถ้าลบผิด) แต่มันจะสร้าง Ticket หรือรายงานไว้ใน **`DQ_Review_Queue`**

4.  **💾 รวมข้อมูลซ้ำด้วยตัวเอง (Merge Duplicates Manual)**
    *   **ใช้งานเมื่อ:** ในคิวแจ้งว่าเจอคนซ้ำ และคุณมั่นใจว่าเป็นบริษัทเดียวกัน
    *   **วิธีใช้:** กดปุ่มนี้ มันจะขอ **Master UUID** (ตัวที่จะเก็บไว้) และ **Duplicate UUID** (ตัวที่จะลบทิ้ง) ระบบจะไม่อนุญาตให้ลบทิ้งจริงๆ (Soft Delete) แต่เปลี่ยน Record_Status ของตัว Duplicate เป็น `Merged`

5.  **🗄️ เก็บกวาดข้อมูลเก่า (Auto Archive INACTIVE) [ฟีเจอร์ใหม่]**
    *   **ใช้งานเมื่อ:** ฐานข้อมูลเริ่มเกินระดับ 10,000 แถว และพบว่ามีลูกค้าบางเจ้า `Inactive` และ `Merged` ไปแล้ว ไม่อยากให้หนักชีตกลาง
    *   **วิธีใช้:** กดปุ่มนี้ ระบบจะอุ้มข้อมูลพวกนั้นไปอยู่ในชีต `Archive_DB` แล้วลบทิ้งออกจากชี้ตหลัก (ช่วยเพิ่มความเร็วแบบทวีคูณ)

## 🤖 3. จัดการ AI Agent (ตั้งค่า API Key)

หากคุณเจออาการ "AI ไม่ทำงาน" หรือ "API Key Invalid" ตามสเปค Enterprise ของระบบ ต้องทำตามนี้:
1. กดเมนู **⚙️ ตั้งค่า (Settings) > 🔑 ตั้งค่า API Key**
2. ใส่ Key ของคุณ (ต้องขึ้นต้นด้วย `AIza` และความยาวเกิน 30 ตัวอักษร)
3. ระบบจะบันทึกลงสู่ Server Properties ป้องกันคนมาแอบขโมยดูจากชีตปกติ

> [!TIP]
> **เคล็ดลับการรันโปรเจคระยะยาว**
> ไม่จำเป็นต้องมานั่งกดวิเคราะห์หรือคลีนด้วยมือทุกวัน ในอนาคตคุณสามารถเข้าไปที่ **Google Apps Script -> Triggers (รูปนาฬิกา)**
> เลือกรันฟังก์ชัน `runDataQualityCheck_Full()` ให้มันทำงานแบบ Time-driven (เช่น ทุกวันอาทิตย์กลางคืน) 
> พอเช้าวันจันทร์ ทีมงานก็แค่เข้ามาเปิดดูว่ามียอดตกค้างใน **DQ_Review_Queue** กี่บรรทัด แล้วเลือกกด "Merge" ทีละตัวให้เสร็จครับ
